# Smart-Home

Учебный проект по дисциплине "Интернет вещей".  
Репозиторий содержит 2 лабораторные работы:
- `лр1` — веб-интерфейс умного дома (frontend);
- `лр2` — Flask-приложение с UML-классами и управлением устройствами через кнопки в вебе.

## Структура проекта

- `лр1/server` — страницы `index.html`, `admin.html`, `user.html`, стили и скрипты ЛР1.
- `лр1/smart_home_classes.puml` — UML-диаграмма классов системы.
- `лр2/things.py` — реализация классов по UML (Python).
- `лр2/app.py` — Flask-сервер и API.
- `лр2/server` — веб-интерфейс ЛР2 (админ/пользователь), работающий через API.

## Лабораторная работа 1 (ЛР1)

### Что реализовано

- Главная страница с переходом в режим:
  - пользователя;
  - администратора (с проверкой роли).
- Два отдельных интерфейса:
  - `admin.html` — полный доступ;
  - `user.html` — ограниченный доступ.
- Управление устройствами (в интерфейсе):
  - чайник (вкл/выкл, целевая температура);
  - климат (температура и влажность);
  - шторы (положение);
  - лампы (яркость и RGB);
  - робот-пылесос (состояние, карта дома).
- Улучшенный UI/UX:
  - тёмная/светлая тема;
  - сводка по кнопке;
  - адаптивная сетка блоков;
  - визуальные индикаторы и сообщения.

### Назначение ЛР1

ЛР1 демонстрирует удобный web-интерфейс "умного дома" и логику ролей без обязательной backend-обработки всех действий.

---

## Лабораторная работа 2 (ЛР2)

### Цель

Реализовать backend на Flask строго по UML-диаграмме и показать работу методов классов через web-интерфейс и API.

### Что реализовано

- Классы по UML в `лр2/things.py`:
  - `SmartDevice` (абстрактный базовый класс),
  - `RobotVacuum`,
  - `SmartCurtains`,
  - `SmartKettle`,
  - `TemperatureControl`,
  - `HumidityControl`,
  - `SmartLighting`,
  - `MainControlUnit`,
  - `Database`,
  - `ChildUI`, `ParentUI`.
- Flask-приложение в `лр2/app.py`:
  - выдача страниц `index/admin/user`;
  - API для управления устройствами;
  - серверное состояние устройств (чайник, шторы, свет, пылесос и т.д.).
- Интерактивное тестирование:
  - кнопки в `лр2/server/*.html` отправляют запросы в Flask API;
  - статус устройств обновляется автоматически.

## Как запустить ЛР2

### 1) Требования

- Python 3.10+ (рекомендуется 3.11+).
- Установленный Flask:

```bash
pip install flask
```

### 2) Переход в папку ЛР2

```bash
cd лр2
```

### 3) Запуск сервера

```bash
python app.py
```

После запуска Flask обычно доступен по адресу:  
`http://127.0.0.1:5000`

### 4) Открытие интерфейса

Открой в браузере:
- `http://127.0.0.1:5000/index.html`

Важно: для ЛР2 открывать страницы нужно именно через Flask (`:5000`), а не как локальный файл и не через Live Server (`:5500`), иначе API работать не будет.

## Как пользоваться ЛР2 (красивый пошаговый сценарий)

### Шаг 1. Выбери роль

На `index.html` перейди в:
- панель администратора (полный контроль),
- или панель пользователя (ограниченный доступ).

### Шаг 2. Управляй устройствами

В админ-панели доступны:
- чайник: включение, выключение, сохранение целевой температуры;
- шторы: открыть, закрыть, сохранить положение;
- лампы: вкл/выкл, яркость, RGB и применение настроек;
- климат: сохранение температуры и влажности;
- пылесос: старт, пауза, возврат на базу, загрузка/просмотр карты.

### Шаг 3. Отслеживай состояние

Статусы в карточках обновляются автоматически (температура чайника, состояние пылесоса, карта, список устройств и т.д.).

### Шаг 4. Проверяй сообщения

После каждого действия показывается сообщение об успехе или ошибке, чтобы сразу видеть результат API-вызова.

## Полезно для отладки

- Проверка API:
  - `http://127.0.0.1:5000/api`
  - `http://127.0.0.1:5000/api/status`
- Если изменения в JS/HTML не видны:
  - обнови страницу с очисткой кэша (`Ctrl+F5`);
  - перезапусти Flask-сервер.
