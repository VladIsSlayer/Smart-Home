# Smart-Home

Учебный проект по дисциплине «Интернет вещей» (тема: **умный дом**).  
Репозиторий содержит лабораторные работы **LR1–LR9**.

| Папка | Содержание |
|-------|------------|
| **LR1** | UML-диаграмма, веб-макеты (статика) |
| **LR2** | Flask + классы Python, REST API (`POST /api/...`) |
| **LR3** | Мониторинг: `GET /connect_*`, опрос датчиков |
| **LR4** | Управление: `GET /control_*` + мониторинг |
| **LR5** | Валидация входящих команд |
| **LR6** | Автоматика `HomeAutomation` (климат → шторы и свет) |
| **LR7** | Логирование MongoDB / JSON |
| **LR8** | Анализ данных (среднее, максимум) |
| **LR9** | Визуализация Chart.js |

## Структура

```
Smart-Home/
├── LR1/          # smart_home_classes.puml, server/ (макеты)
├── LR2/          # things.py, app.py, server/
├── LR3–LR9/      # things.py, app.py, server/, task.txt
├── tools/        # generate_report_guides.py, sync_device_fixes.py
├── requirements.txt
└── Методичка.txt
```

В каждой LR4–LR9 есть файл **`Otchet_LR{N}_soderzhanie.docx`** — что включать в отчёт.

## Запуск (LR3–LR9)

```bash
pip install -r requirements.txt
cd LR4
py -3 app.py
```

Браузер: **http://127.0.0.1:5000/index.html** (только через Flask, не `file://`).

- Пользователь: кнопка на главной странице  
- Администратор: PIN **`1234`**

Одновременно может работать только одна ЛР (порт 5000).

## Поведение устройств (актуально с LR4)

| Устройство | Мониторинг | Управление |
|------------|------------|------------|
| **Шторы** | Положение стабильно; редкий дрейф ±1% раз в **2 мин** | «Открыть» → 100%, «Закрыть» → 0%, «Сохранить» → значение слайдера |
| **Лампы** | Яркость и **RGB не меняются** сами | «Применить RGB» сохраняет r, g, b на сервере |
| **Климат** | Текущие T и влажность **плавно стремятся к цели** (~0.2°C / 0.5% каждые 20 с) | Поля **целевых** значений; кнопка «Применить цели климата» |
| Чайник, пылесос | Эмуляция по методичке | Кнопки `/control_*` |

## LR7–LR9: база данных

- При наличии **MongoDB** (`localhost:27017`) — коллекции `Temperature`, `Humidity` в БД `IOT_logger_db`.
- Иначе — файл `LR7/data/IOT_logger_db.json` (аналогично в LR8/LR9 при запуске из своих папок).
